Структура геолого-промысловой базы данных

Структура геолого-промысловой базы данных заимствована у продукта Finder фирмы Slumberger с небольшими изменениями. В настоящее время эта БД служит также и единственным источником данных для импорта в Field. При импорте делается попытка исправить отдельные ошибки, которые присутствуют в БД Finder. В текущей реализации база данных хранится в формате MDB (Microsoft Jet Database Engine), позволяющим с одной сторона извлекать информацию с помощью языка SQL, а с другой стороны не требующим установки отдельного сервера для обслуживания базы данных. Всвязи с ограничением в 2Gb, налогаемым на размер базы данных, для хранения массивов информации используется сопутствующий файл в формате HDF (Hierarchical Data Format, National Center for Supercomputing Applications). В базе данных используются следующие таблицы:

Текстовая информация
События на скважинах
Интерпретация ГИС
История работы скважин
Библиотека каротажных кривых
Интервалы перфорации
Инклинометрия скважин
Общая информация по скважинам
Кустование скважин

Также она может использовалься как источник данных по информации о выделенных пропластках. Ввиду большого объёма исходных данных, информация по пропласткам хранится в разделе /geophysics/uwi файла формата HDF в компактном виде. Инклинометрия скважин

Массивы, описывающие геометрию ствола скважины хранятся в сопутствующем файле формата HDF по адресу

/traj/<версия замера>/<идентификатор скважины>.

Каждый элемент массива содержит картеж чисел с плавающей точкой {dx, dy, dz; md}, который сопоставляет расстоянию md, измеренному вдоль ствола скважины, относительное смещение точки замера (dx, dy, dz) от устья скважины. Записи таблицы traj базы данных программы ссылаются на эти массивы. Таблица проиндексирована ключём bore_id, который содержит поля (traj_wb, trj_id). Кроме того, для строго вертикальных скважин достаточно иметь только запись в таблице traj, которая полностью определяет геометрию ствола скважины. Таблица имеет следующий вид:

НомерИмяТипДлинаЗначение по умолчаниюКомментарий
1traj_wbDBTYPE_I4  универсальный идентификатор скважины
2traj_idDBTYPE_UI1  идентификатор замера инклинометрии
3traj_dxDBTYPE_R4  отклонение забоя по направлению запад-восток
4traj_dyDBTYPE_R4  отклонение забоя по направлению юг-север
5traj_dzDBTYPE_R4  абсолютная глубина от поверхности стола ротора
6traj_mdDBTYPE_R4  глубина, на которой прерван замер инклинометрии
6traj_nsDBTYPE_I2  число элементов в массиве замера инклинометрии

Кустование скважинам

В процессе эксплуатации программы было выявлено, что координаты устья скважин, заданные в базе данных Finder, отстоят на большом удалении от куста скважин (десятки километров), которому они принадлежат, нарушая тем самым ковёр разбуривания. Данной программой такая ситуация трактуется как ошибка и применяется специальный алгоритм для автоматической корректировки положения скважин на основе кластирезации. На следующей иллюстрации отражена работа алгоритма по восстановлению ковра разбуривания месторождения, задуманного проектировщиками. Цветом выделены кусты, в состав которых входят скважины, отстоящие на большое расстояние от других скважин, принадлежащих данному кусту.
